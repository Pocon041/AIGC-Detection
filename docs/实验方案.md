# OKK 实验方案：条件扩散低层残差一致性检测

## 0. 目标定位

本实验不是做一个普通的 AIGC 二分类器，而是验证一个更明确的假设：

> 在高质量生成图、语义对齐、跨生成器和强后处理场景下，单纯语义分类或静态低层密度建模不稳定；更有价值的方向是先学习对低层像素级变化敏感的特征，再在内容条件下判断低层残差是否符合真实摄影图像的自然分布。

方法暂定名：

> Conditional Diffusion Low-level Residual Learning，简称 CDL-RL。

中文名：

> 条件扩散低层残差学习。

最终目标不是只在同分布数据上刷 accuracy，而是尽量做到：

1. 跨生成器泛化更强。
2. 跨操作泛化更强。
3. 后处理鲁棒性更强。
4. 能输出 patch-level suspicious heatmap。
5. 相比 Beyond Generation-style 边际低层分布建模，证明条件建模有额外收益。

需要先说明：SOTA 是目标，不是代码一写就能保证。这个项目必须按可验证实验推进，不能用复杂结构掩盖信号本身是否存在。

---

## 1. 已知经验约束

根据前面实验总结，本项目要吸收以下经验：

### 1.1 有效方向

1. 扰动响应优于静态密度。
   - 对图像施加受控退化后观察特征变化，比直接建模特征分布更有效。
   - 因此本项目不能只做 `p_real(z)` 的 GMM / KDE / one-class density。

2. DINOv2 冻结特征是强先验。
   - from-scratch encoder 在小数据和弱信号下很容易失败。
   - 因此结构特征 `C` 或扰动响应主干应优先使用冻结 DINOv2 / ConvNeXt / CLIP 作为基线。

3. 噪声退化响应是强跨域信号。
   - 微弱高斯噪声下的特征偏移曾经在 Chameleon 上达到较高 AUROC。
   - 因此 `F_low` 不应只输出静态低层特征，还应支持 multi-level noise response profile。

4. Patch 级统计优于全局 mean。
   - patch-level q90、lower-tail、variance 可能比整图平均更有信息。
   - 因此模型必须保留 patch-level map，而不是一开始就全局池化。

### 1.2 应避免的方向

1. 不把 from-scratch 像素级 NLL 当主线。
2. 不依赖简单跨层预测器，因为残差连接会让预测任务过易。
3. 不把高维中间层静态密度建模当主线，弱信号容易被噪声维度淹没。
4. 不默认 GMM 融合有效，弱信号可能淹没强信号。融合优先用显式 z-score、logistic calibration 或小型验证集学习权重。
5. 不把 MAE 重建误差作为核心检测信号。
6. 不用 AMP 混合精度。

## 2. 核心假设

本项目要验证三个层层递进的假设。

### H1：低层预训练特征比普通语义特征更适合鉴伪

普通 CLIP / DINO / ImageNet CNN 主要服务于语义识别，对微弱低层差异可能不敏感。

我们需要训练或构造一个低层特征提取器 `F_low`，使它对以下差异敏感：

1. diffusion denoise 前后差异。
2. 微弱高斯噪声扰动响应。
3. 局部纹理和噪声统计变化。
4. patch-level 低层自然性变化。

### H2：条件低层残差建模优于边际低层分布建模

Beyond Generation-style 方法主要建模：

```text
p_real(R)
```

其中：

```text
R = F_low(x)
```

本项目希望推进为：

```text
p_real(R | C)
```

其中：

```text
C = content / structure / low-frequency condition
R = low-level residual feature
```

核心问题是：

> 在给定内容结构 `C` 的情况下，当前低层残差 `R` 是否像真实摄影图产生的？

### H3：patch-level 分布式证据比少数 patch 或全局 mean 更稳健

高质量 AIGC 的痕迹可能弱而分散。局部 AI 编辑的痕迹可能局部集中。

因此模型应该输出：

```text
patch_score_map
```

再聚合为：

```text
image_score = mean_score + lower_tail_penalty + variance_term
```

这样同时支持：

1. 整图 AIGC 检测。
2. 局部 AI 编辑检测。
3. suspicious heatmap 可视化。

## 3. 方法整体结构

完整系统分为四个模块。

### 3.1 `F_low`：低层特征提取器

输入：

```text
x: image
```

输出：

```text
R = F_low(x): patch-level low-level feature map
```

训练或构造方式分三版递进。

#### V0：冻结 DINOv2 扰动响应基线

这是最快可跑的强基线。

对输入图像 `x` 构造多种扰动：

```text
x
x + Gaussian noise sigma=0.005
x + Gaussian noise sigma=0.010
x + Gaussian noise sigma=0.020
JPEG(x, q=95/85)
blur(x, sigma=0.5/1.0)
```

送入冻结 DINOv2，提取 patch token，计算响应特征：

```text
cos(F(x), F(T(x)))
L2(F(x), F(T(x)))
patch-wise delta norm
q90 / q10 / std / mean
```

优点：

1. 快。
2. 和已有经验一致。
3. 不需要 diffusion 预处理数据。
4. 可作为所有复杂方法的下限基线。

#### V1：Beyond Generation-style diffusion pretext `F_low`

只用真实图训练。

对真实图 `x`：

```text
x -> add noise at timestep t -> diffusion denoise -> x_denoised_t
```

训练 `F_low` 区分：

```text
original real image
vs
diffusion-denoised variant
```

目标不是检测 AIGC，而是迫使 encoder 学会捕捉 pixel-level low-level difference。

训练任务可选：

1. image-level classification：原图 vs denoised。
2. patch-level classification：每个 patch 判断是否来自 denoised variant。
3. contrastive：同一图不同 denoise level 形成有序 low-level perturbation embedding。

输出保留 patch feature map。

#### V2：diffusion pretext + perturbation response `F_low`

在 V1 基础上加入显式扰动响应谱。

对图像 `x`：

```text
R_static = F_low(x)
R_noise = F_low(x) - F_low(noise(x))
R_jpeg = F_low(x) - F_low(jpeg(x))
R_blur = F_low(x) - F_low(blur(x))
```

最终低层残差特征：

```text
R = concat(R_static, R_noise, R_jpeg, R_blur)
```

这个版本更符合已有结论：扰动响应优于静态密度。

---

### 3.2 `C_encoder`：内容 / 结构条件提取器

输入：

```text
x: image
```

输出：

```text
C = C_encoder(x): patch-level or low-resolution condition feature
```

`C` 的要求：

1. 保留内容、布局、低频结构。
2. 尽量去掉高频纹理、噪声和生成器 artifact。
3. 单独用 `C` 做 real/fake 不应太强，否则会退化为普通分类器。

候选实现：

#### C0：低频图像特征

```text
x_low = GaussianBlur(x)
x_low = downsample_upsample(x)
C = shallow CNN(x_low)
```

优点：解释干净。缺点：表达能力有限。

#### C1：冻结 DINOv2 / CLIP 语义结构特征

```text
C = frozen_backbone(x).patch_tokens
```

优点：强先验。缺点：可能包含部分纹理和域偏差。

#### C2：低频图像 + 冻结语义特征融合

```text
C = concat(C_lowfreq, C_dino)
```

这是推荐主线。

必须做的控制实验：

```text
only C classifier
```

如果 `only C` 已经很强，说明数据存在内容偏差或条件特征泄露低层伪影。

---

### 3.3 `D_theta`：条件残差自然性打分器

输入：

```text
C: content condition map
R: low-level residual map
```

输出：

```text
patch_scores: 每个 patch 的真实自然性分数
image_score: 聚合后的图像真实自然性分数
```

基本形式：

```text
H = fuse(C, R)
patch_scores = MLP_or_Transformer(H)
```

推荐从简单模型开始：

1. channel concat + 1x1 conv。
2. shallow transformer over patch tokens。
3. FiLM conditioning：用 `C` 调制 `R`。
4. cross-attention：`C` as query，`R` as key/value。

第一版不要过度复杂。优先用：

```text
concat(C, R) -> 2-layer MLP -> patch_score
```

因为如果简单版本完全无效，复杂版本大概率只是过拟合。

---

### 3.4 Aggregator：patch 分数聚合

patch score 越高表示越像真实摄影图。

图像级分数：

```text
mean_score = mean(patch_scores)
lower_tail = mean(k lowest patch_scores)
score_std = std(patch_scores)
image_score = mean_score + alpha * lower_tail - beta * score_std
```

需要验证不同聚合：

1. mean。
2. q10。
3. q25。
4. q90。
5. mean + lower-tail。
6. learned logistic aggregator。

整图 AIGC 检测一般看 mean / q25。
局部 AI 编辑检测一般看 lower-tail / min-k。

## 4. 数据组织

项目建议使用统一 manifest，而不是在代码里写死目录。

### 4.1 manifest 格式

每一行表示一个样本：

```text
path,label,group,pair_id,mask_path,operation,generator,split
```

字段含义：

```text
path: 图像路径
label: 0=real, 1=fake
group: 数据集名称，如 genimage/chameleon/semitruths/bfree/dda_coco
pair_id: 语义对齐 pair 的 ID，没有则为空
mask_path: 局部编辑 mask，没有则为空
operation: full_generation/inpainting/reconstruction/local_edit/super_resolution
generator: SDXL/FLUX/Midjourney/GAN/real 等
split: train/val/test
```

### 4.2 训练数据优先级

#### 第一优先级：真实图

用于训练 `F_low` pretext。

可用：

1. ImageNet real。
2. COCO real。
3. DIV2K / Flickr2K 高质量真实图。
4. GenImage real subset。

注意事项：

1. 真实图格式要统一。
2. JPEG/PNG 混用可能造成伪影。
3. 不能让 real/fake 的压缩格式成为标签泄露。

#### 第二优先级：语义对齐 real/fake pair

用于训练条件残差判别器。

可用：

1. B-Free-style self-conditioned fake。
2. Semi-Truths 局部 AI augmentation。
3. 自己用 diffusion reconstruction / inpainting 构造 pair。

要求：

1. `x_real` 和 `x_fake_aligned` 内容接近。
2. 低频结构接近。
3. fake 操作类型要记录。
4. 如果有 mask，要保存局部监督。

#### 第三优先级：跨生成器测试集

用于最终泛化评估。

可用：

1. GenImage。
2. Chameleon。
3. DDA-COCO。
4. DRCT-2M subset。
5. FLUX / SDXL / Midjourney / commercial generators，如果有。

## 5. 训练流程

### Stage 0：建立强基线

目的：先验证数据和评估流程健康。

模型：

```text
DINOv2 perturbation response baseline
```

训练：

```text
不训练 backbone
只训练 logistic regression / small MLP
```

输入特征：

```text
noise response
jpeg response
blur response
patch statistics
```

评估：

```text
AUROC
AP
ACC
TPR@FPR=1%
TPR@FPR=5%
```

如果这个 baseline 都跑不通，不进入复杂模型。

---

### Stage 1：训练 `F_low`

输入：真实图。

生成 diffusion-denoised variants：

```text
x_denoised_t1
x_denoised_t2
x_denoised_t3
```

训练标签：

```text
0: original real
1: denoised variant
```

推荐损失：

```text
L_pretext = BCE(image_logits, y) + lambda_patch * BCE(patch_logits, y_patch)
```

如果设计有 timestep 分类：

```text
L_timestep = CE(timestep_logits, t)
```

总损失：

```text
L = L_pretext + lambda_t * L_timestep
```

训练完成后，保存：

```text
checkpoints/f_low_best.pth
```

并冻结 `F_low` 进入 Stage 2。

---

### Stage 2：训练条件残差判别器

输入 pair：

```text
x_real
x_fake_aligned
```

提取：

```text
C_real = C_encoder(x_real)
C_fake = C_encoder(x_fake_aligned)
R_real = F_low(x_real)
R_fake = F_low(x_fake_aligned)
```

主要训练方式有两种。

#### 方式 A：pair ranking loss

```text
s_real = D(C_real, R_real)
s_fake_anchor = D(C_real, R_fake)
s_fake_self = D(C_fake, R_fake)
```

损失：

```text
L_rank = max(0, margin - s_real + s_fake_anchor)
       + max(0, margin - s_real + s_fake_self)
```

优势：直接体现“同内容下真实残差更自然”。

#### 方式 B：patch-level BCE

```text
real patches -> label 1
fake patches -> label 0
```

如果有 mask：

```text
modified fake patches -> label 0
unmodified fake patches -> ignore or weak label 1
```

损失：

```text
L_patch = BCE(patch_scores, patch_labels)
```

最终推荐：

```text
L_cond = L_rank + lambda_patch * L_patch
```

---

### Stage 3：operator-invariance 训练

目的：避免模型记住某个构造器的指纹。

假设 fake 有操作类型：

```text
diffusion_reconstruction
inpainting
vae_reconstruction
super_resolution
local_edit
```

增加 operator adversarial head：

```text
operator_logits = OpHead(feature)
```

训练逻辑：

1. `OpHead` 尽量预测 operation。
2. 主干通过 gradient reversal 让 operation 不可预测。
3. 主任务仍然区分真实自然残差和异常残差。

损失：

```text
L = L_cond - lambda_adv * L_operator
```

或者用 Gradient Reversal Layer 实现：

```text
L = L_cond + lambda_adv * CE(operator_logits, operation)
```

其中 GRL 会自动反转主干梯度。

第一版可以先不启用 adversarial，只保留接口。等主信号有效后再加。

---

## 6. 测试流程

给定未知图像 `y`：

```text
C_y = C_encoder(y)
R_y = F_low(y)
patch_scores = D(C_y, R_y)
image_score = aggregate(patch_scores)
```

输出：

1. image-level realness score。
2. image-level suspicious score。
3. real/fake decision。
4. patch-level suspicious heatmap。

分数方向统一：

```text
realness_score 越高越真实
suspicious_score = -realness_score
suspicious_score 越高越可疑
```

所有评估脚本必须显式打印分数方向，避免 AUROC 方向反了还误判。

## 7. 实验矩阵

### 7.1 实验 A：验证 `F_low` 必要性

对比：

```text
CLIP feature
DINOv2 feature
DINOv2 perturbation response
hand-crafted DCT/SRM/HOG
Beyond Generation-style F_low
F_low + perturbation response
```

指标：

```text
AUROC
AP
TPR@FPR=1%
TPR@FPR=5%
```

预期：

```text
F_low + response > DINOv2 static > hand-crafted
```

如果 `F_low` 不如 DINOv2 response，说明 diffusion pretext 没有带来有效增益，需要回到 Stage 0 强基线。

---

### 7.2 实验 B：验证 conditional modeling 有增益

对比：

```text
Marginal: score = p_real(R)
Conditional: score = D(C, R)
Only R classifier
Only C classifier
C + R classifier without pair ranking
```

目标：

证明 `D(C, R)` 不是普通特征拼接分类器，而是在内容条件下建模残差自然性。

成功标准：

1. Conditional > Marginal。
2. Conditional > Only R。
3. Only C 不应过强。
4. 跨生成器测试 Conditional 优势更明显。

---

### 7.3 实验 C：语义对齐训练是否有效

对比训练数据：

```text
普通 real/fake unpaired training
semantic-aligned pair training
semantic-aligned pair + ranking
semantic-aligned pair + ranking + patch loss
```

测试：

```text
同分布
跨生成器
跨数据集
跨操作
```

如果 aligned training 只提升同分布，不提升跨域，说明它可能只是带来了新的偏差。

---

### 7.4 实验 D：跨生成器泛化

训练：

```text
SD / inpainting / reconstruction subset
```

测试：

```text
SDXL
FLUX
Midjourney
Wukong
GAN
flow-based models
commercial generators
```

重点指标：

```text
TPR@FPR=1%
```

因为实际鉴伪不能接受高误报。

---

### 7.5 实验 E：跨操作泛化

训练 / 测试拆分示例：

```text
train: diffusion reconstruction
test: inpainting, local edit, VAE reconstruction, super-resolution
```

反向也要做：

```text
train: local edit
test: full-image generation
```

目标：

证明模型没有记住某个 fake construction operator 的指纹。

---

### 7.6 实验 F：局部检测能力

数据：

```text
Semi-Truths 或其他带 mask 的局部编辑数据
```

指标：

```text
image-level AUROC
patch-level AUROC
mask IoU
pixel/patch localization AP
```

输出：

```text
suspicious heatmap
```

如果 image-level 有效但 localization 无效，需要重新检查 patch 聚合和 patch 监督。

---

### 7.7 实验 G：后处理鲁棒性

测试后处理：

```text
JPEG q=95/85/75/65
resize 0.5x/0.75x/1.5x
Gaussian blur sigma=0.5/1.0
crop + resize
color jitter
screenshot-like resampling
social media recompression
```

每种后处理都输出：

```text
AUROC
AP
TPR@FPR=1%
```

如果 JPEG 后性能大幅崩溃，需要判断模型是否过度依赖脆弱高频噪声。

---

### 7.8 实验 H：完整消融

必须做：

```text
w/o diffusion pretext
w/o perturbation response
w/o conditional C
w/o semantic-aligned pairs
w/o pair ranking loss
w/o patch-level loss
w/o operator adversarial
w/o lower-tail aggregation
only image-level classifier
only one-class GMM
only DINOv2 static
only DINOv2 response
```

消融目的：

1. 证明 `F_low` 不是普通 CNN。
2. 证明 `C` 条件有用。
3. 证明 aligned pair 不是单纯数据增强。
4. 证明 patch-level 不是装饰。
5. 证明 operator adversarial 没有副作用，或者确实提升跨操作泛化。

---

## 8. 实现顺序

### Phase 1：最小强基线

目标：不用 diffusion，先跑通强基线。

实现文件：

```text
config.py
dataset.py
backbones.py
transforms.py
features_dino_response.py
train_probe.py
evaluate.py
metrics.py
```

功能：

1. manifest 数据读取。
2. 冻结 DINOv2 特征。
3. 多扰动响应特征。
4. logistic / MLP probe。
5. AUROC、AP、TPR@FPR。

成功标准：

```text
能在一个小数据集上完成 train/val/test，并输出稳定指标。
```

---

### Phase 2：条件残差框架

实现文件：

```text
condition_encoder.py
lowlevel_encoder.py
conditional_detector.py
train_conditional.py
evaluate_conditional.py
visualize_heatmap.py
```

功能：

1. `C_encoder`。
2. `F_low` 初版可先用 DINOv2 response feature。
3. `D_theta(C, R)`。
4. pair ranking loss。
5. patch score map。
6. heatmap 输出。

成功标准：

```text
Conditional > Only R 或至少不低于 Only R。
```

如果不成立，先不要进入 diffusion pretext。

---

### Phase 3：diffusion pretext `F_low`

实现文件：

```text
prepare_diffusion_variants.py
train_f_low.py
extract_f_low.py
```

功能：

1. 对真实图生成 diffusion-denoised variants。
2. 训练 `F_low` 区分 original vs denoised。
3. 输出 patch feature map。
4. 替换 Phase 2 中的 DINOv2 response `R`。

成功标准：

```text
F_low conditional >= DINOv2 response conditional
```

如果 diffusion pretext 不如 DINOv2 response，说明 pretext 或模型设计需要调整。

---

### Phase 4：operator adversarial 和局部监督

实现：

1. Gradient Reversal Layer。
2. operator head。
3. mask-aware patch loss。
4. local editing evaluation。

成功标准：

```text
跨操作 holdout 提升，且同分布性能不明显下降。
```

## 11. 风险点和对应检查

### 风险 1：模型退化为普通监督分类器

检查：

```text
only C
only R
C + R without ranking
cross-generator holdout
cross-operation holdout
```

### 风险 2：`C` 泄露低层伪影

检查：

```text
only C AUROC
blurred C only
low-frequency C only
C feature visualization
```

### 风险 3：`F_low` 学到 denoiser 指纹

检查：

```text
不同 denoiser / timestep holdout
JPEG / resize robustness
train on denoised real, test on real AIGC
```

### 风险 4：低层特征被后处理摧毁

检查：

```text
JPEG q=95/85/75/65
resize
blur
social-media recompression
```

### 风险 5：融合方式淹没强信号

检查：

```text
单信号结果
z-score weighted sum
learned logistic calibration
GMM fusion 对照
```

### 风险 6：同分布好，跨域差

检查：

```text
按 generator / dataset / operation 分组输出指标
不能只报告平均 AUROC
```

## 12. 预期结论形式

如果方法成功，最终应该能得到如下结论：

1. `F_low` 或 perturbation response 明显优于静态语义特征。
2. 条件建模 `D(C, R)` 在跨生成器或跨操作上优于边际建模 `p(R)`。
3. patch-level aggregation 在局部编辑和整图检测上均有收益。
4. operator adversarial 能提升跨操作泛化，或至少减少对某个构造器的依赖。
5. 强后处理下性能下降可控。

如果方法失败，也要明确失败类型：

1. 低层信号本身不足。
2. 条件信息没有带来增益。
3. 信号主要来自数据格式偏差。
4. 后处理破坏所有低层特征。
5. DINOv2 response 已经接近上限，diffusion pretext 没有新增价值。

失败也有价值，因为可以避免继续在错误方向上堆复杂模型。

## 13. 当前最推荐执行路线

优先顺序如下：

```text
1. 先做 manifest + metrics + DINOv2 response baseline。
2. 再做 only R / only C / C+R 三个 probe。
3. 再做 pair ranking conditional detector。
4. 再做 patch aggregation 和 heatmap。
5. 如果 conditional 有增益，再做 diffusion pretext F_low。
6. 最后做 operator adversarial、局部监督和完整鲁棒性评估。
```

这条路线的好处：

1. 每一步都有明确可证伪指标。
2. 不会一开始就陷入 diffusion 预处理成本。
3. 能充分利用已有结论：扰动响应、DINOv2、patch 统计。
4. 如果最终上 diffusion pretext，也有强基线判断它是否真的贡献了增益。
