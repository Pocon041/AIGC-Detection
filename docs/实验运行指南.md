# OKK 精简实验运行指南

## 1. 主线目标

本项目分两步验证 AIGC 检测思路：

1. 先按 Beyond Generation 论文训练低级特征提取器 `F_low`。
2. 再用 DINOv3/低频内容作为条件 `C`，统计或学习 `p(F_low | C)`。

关键原则：

- DINOv3 只作为高级内容条件编码器，不作为低级特征提取器。
- Beyond 低级特征提取器必须用预计算的 `original + vae_recon + diffusion-denoised variants` 训练。
- 不使用 AMP 混合精度。

## 2. 关键文件

| 文件 | 作用 |
|---|---|
| `prepare_beyond_precomputed.py` | 用 Stable Diffusion v1.4 生成 Beyond 预训练变体 |
| `train_lowlevel_precomputed.py` | 训练论文版 Beyond `F_low` |
| `fit_beyond_gmm.py` | 论文式 one-class GMM 检测 |
| `make_dda_coco_manifest.py` | 生成 DDA-COCO 普通/配对 manifest |
| `train_conditional.py` | 训练条件残差检测器 |
| `evaluate_conditional.py` | 评估条件残差检测器 |
| `visualize_heatmap.py` | 生成 patch suspicious heatmap |

不要使用：

| 文件 | 原因 |
|---|---|
| `train_lowlevel_pretext.py` | 旧的在线 blur/noise 近似入口，已禁用 |

## 3. 安装依赖

在 `d:\OKK` 项目目录下运行：

```powershell
python -m pip install -r requirements.txt
```

如果下载 Stable Diffusion v1.4 需要 HuggingFace 权限，请先在本机完成登录或准备本地模型缓存。

### 3.1 Linux/AutoDL 显式下载 Stable Diffusion v1.4

论文版 Beyond 预计算必须使用 diffusion denoising，不能用 blur 近似替代。

以下命令适用于 Linux/AutoDL，模型下载到 `~/autodl-tmp/stable-diffusion-v1-4`。

先进入项目目录并安装依赖：

```bash
cd ~/OKK
python -m pip install -r requirements.txt
```

如果 `CompVis/stable-diffusion-v1-4` 需要 HuggingFace 权限，先登录：

```bash
huggingface-cli login
```

显式下载模型到本地目录：

```bash
mkdir -p ~/autodl-tmp/stable-diffusion-v1-4
huggingface-cli download CompVis/stable-diffusion-v1-4 \
  --local-dir ~/autodl-tmp/stable-diffusion-v1-4 \
  --local-dir-use-symlinks False
```

先小规模测试 10 张：

```bash
python prepare_beyond_precomputed.py \
  --manifest manifests/dda_coco_manifest.csv \
  --split train \
  --model ~/autodl-tmp/stable-diffusion-v1-4 \
  --out-dir data/beyond_precomputed/debug \
  --out-manifest manifests/beyond_precomputed_debug.csv \
  --max-images 10
```

确认 `data/beyond_precomputed/debug` 下的 `original/vae_recon/denoise_t*` 正常后，再跑完整 train：

```bash
python prepare_beyond_precomputed.py \
  --manifest manifests/dda_coco_manifest.csv \
  --split train \
  --model ~/autodl-tmp/stable-diffusion-v1-4 \
  --out-dir data/beyond_precomputed/train \
  --out-manifest manifests/beyond_precomputed_train.csv \
  --max-images 12000
```

验证集：

```bash
python prepare_beyond_precomputed.py \
  --manifest manifests/dda_coco_manifest.csv \
  --split val \
  --model ~/autodl-tmp/stable-diffusion-v1-4 \
  --out-dir data/beyond_precomputed/val \
  --out-manifest manifests/beyond_precomputed_val.csv \
  --max-images 2000
```

## 4. 生成 DDA-COCO manifest

### 4.1 普通评估 manifest

用于 GMM 或条件模型最终评估：

```powershell
python make_dda_coco_manifest.py `
  --real-dir D:/COCO/val2017 `
  --fake-root D:/DDA-COCO/DDA-COCO `
  --generators dalle2,dalle3,sdxl,stable-diffusion-1-5,stable-diffusion-2-1 `
  --train-ratio 0.6 `
  --val-ratio 0.2 `
  --out manifests/dda_coco_manifest.csv
```

如果 generator 名字和你本地目录不一致，先查看：

```powershell
Get-ChildItem D:/DDA-COCO/DDA-COCO -Directory
```

### 4.2 配对训练 manifest

用于 `train_conditional.py`：

```powershell
python make_dda_coco_manifest.py `
  --real-dir D:/COCO/val2017 `
  --fake-root D:/DDA-COCO/DDA-COCO `
  --generators dalle2,dalle3,sdxl,stable-diffusion-1-5,stable-diffusion-2-1 `
```

## 5. 生成 Beyond 预计算变体

`F_low` 预训练只需要真实图。这里从普通 DDA-COCO manifest 中取 `label=0` 的真实图。

```powershell
python prepare_beyond_precomputed.py `
  --manifest manifests/dda_coco_manifest.csv `
  --split train `
  --out-dir data/beyond_precomputed/train `
  --out-manifest manifests/beyond_precomputed_train.csv `
  --max-images 12000
```

验证集：

```powershell
python prepare_beyond_precomputed.py `
  --manifest manifests/dda_coco_manifest.csv `
  --split val `
  --out-dir data/beyond_precomputed/val `
  --out-manifest manifests/beyond_precomputed_val.csv `
  --max-images 2000
```

## 6. 训练 Beyond 低级特征提取器

```powershell
python train_lowlevel_precomputed.py `
  --manifest manifests/beyond_precomputed_train.csv `
  --val-manifest manifests/beyond_precomputed_val.csv `
  --train-split train `
  --val-split val `
  --epochs 30 `
  --batch-size 64 `
  --lr 1e-5 `
  --out checkpoints/beyond_lowlevel_precomputed_best.pth
```

输出：

```text
checkpoints/beyond_lowlevel_precomputed_best.pth
checkpoints/beyond_lowlevel_precomputed_best.json
```

## 7. 先跑论文式 GMM 检测

这一步用于检查 `F_low` 本身有没有学到可用低级分布。

```powershell
python fit_beyond_gmm.py `
  --checkpoint checkpoints/beyond_lowlevel_precomputed_best.pth `
  --calib-manifest manifests/dda_coco_manifest.csv `
  --calib-split train `
  --eval-manifest manifests/dda_coco_manifest.csv `
  --eval-split test `
  --components 6 `
  --threshold-percentile 0.05 `
  --out outputs/beyond_gmm_eval.json `
  --gmm-out checkpoints/beyond_gmm.joblib
```

重点看：

- `overall.auroc`
- `overall.ap`
- `threshold_acc`
- `false_alarm`
- `recall`
- `grouped_by_generator`

如果这一步很差，先不要训练条件模型，应回头检查预计算变体和 `F_low` 训练。

## 8. 训练条件残差检测器

```powershell
python train_conditional.py `
  --manifest manifests/dda_coco_paired.csv `
  --train-split train `
  --val-split val `
  --condition hybrid `
  --residual beyond `
  --beyond-checkpoint checkpoints/beyond_lowlevel_precomputed_best.pth `
  --epochs 20 `
  --batch-size 32 `
  --lr 1e-4 `
  --out checkpoints/conditional_beyond_best.pth
```

输出：

```text
checkpoints/conditional_beyond_best.pth
checkpoints/conditional_beyond_best.json
```

## 9. 评估条件模型

```powershell
python evaluate_conditional.py `
  --manifest manifests/dda_coco_manifest.csv `
  --split test `
  --checkpoint checkpoints/conditional_beyond_best.pth `
  --batch-size 32 `
  --out outputs/conditional_beyond_eval.json
```

输出：

```text
outputs/conditional_beyond_eval.json
outputs/conditional_beyond_eval.npz
```

重点比较：

- `fit_beyond_gmm.py` 的无条件 GMM 结果
- `evaluate_conditional.py` 的条件残差结果

如果条件模型有效，应看到跨 generator 的 AUROC/AP 更稳，或 false alarm 更低。

## 11. 最小完整实验顺序

推荐严格按这个顺序跑：

```text
1. make_dda_coco_manifest.py 生成普通 manifest
2. make_dda_coco_manifest.py --paired 生成 paired manifest
3. prepare_beyond_precomputed.py 生成 train variants
4. prepare_beyond_precomputed.py 生成 val variants
5. train_lowlevel_precomputed.py 训练 F_low
6. fit_beyond_gmm.py 检查 F_low 无条件 one-class 效果
7. train_conditional.py 训练条件残差检测器
8. evaluate_conditional.py 评估条件模型
9. visualize_heatmap.py 做可解释性检查
```

## 12. 成功判据

最低要求：

- Beyond pretext 验证集 AUROC 明显高于随机。
- GMM 检测中 fake likelihood 整体低于 real。
- 条件模型相较 GMM 在跨 generator 上更稳。
- heatmap 不应完全随机，应在纹理异常或生成痕迹区域有响应。

如果失败，优先排查：

1. Stable Diffusion 预计算变体是否正常。
2. `denoise_t1..t9` 与 `original` 是否只有低层差异而非严重语义变化。
3. `F_low` 是否过拟合 pretext 分类但无法泛化到 real/fake。
4. DDA-COCO 的真实图 JPEG 压缩是否导致真实低层分布本身偏移。
